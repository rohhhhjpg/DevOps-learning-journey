These are the phases of SDLC. Where it involves
1.Planning - Important phase Business analyst or some role collect the requirements from the customer and moves to next phase

2.Defining - Produce a Software requirement document which has a proper documentation of what exactly the application has to be and moves to next phase

3.Designing - Here low level and High level Designs were Developed and moves to the next phase.

(This is where the real automation starts and where DevOps gets involved.)
<img width="1100" height="702" alt="image" src="https://github.com/user-attachments/assets/1f5608d7-9283-401f-915f-cc0ce62e73d2" />

4.Building - Developing phase. Developer starts writing the code in centralized repository (GIT). 

5.Testing - Testing phase , where the code launches in server and taken care by quality assurance QE team and next moves to production.

6.Deploy- Production server Codes deployed to production and customers can start using the application.

DevOps plays a role in fastening the process without manual intervention and every process in these phase will be automated and quickly delivered to the customers.

Every organization follows Agile methodology.
